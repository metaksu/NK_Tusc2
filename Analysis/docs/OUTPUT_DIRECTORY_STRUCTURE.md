# Output Directory Structure

This document describes the output directory structure generated by the NK cell analysis pipeline.

## Master Output Directory
```
Combined_NK_TUSC2_Analysis_Output/
```

## Main Analysis Directories

### 0_Setup_Figs
- **Purpose**: Initial setup and validation plots
- **Contents**: PCA variance ratio plots for data quality assessment

### 1_Processed_Anndata  
- **Purpose**: Stores processed AnnData objects for reuse
- **Contents**: 
  - `adata_blood_processed.h5ad`
  - `adata_normal_tissue_processed.h5ad` 
  - `adata_tumor_tissue_processed.h5ad`

### 2_Blood_NK_Char
- **Purpose**: Comprehensive characterization of blood NK cells (Rebuffet dataset)
- **Subdirectories**:
  - `figures/` - All plots and visualizations
  - `data_for_graphpad/` - CSV files for external analysis
  - `stat_results_python/` - Statistical analysis results
  - `marker_lists/` - Gene marker lists for each analysis

### 3_NormalTissue_NK_Char
- **Purpose**: Characterization of normal tissue NK cells (Tang dataset)
- **Structure**: Same as Blood_NK_Char

### 4_TumorTissue_NK_Char  
- **Purpose**: Characterization of tumor tissue NK cells (Tang dataset)
- **Structure**: Same as Blood_NK_Char

### 5_TUSC2_Analysis
- **Purpose**: Multi-layered TUSC2 expression analysis
- **Subdirectories**:
  - `5A_Broad_Context/` - Cross-context TUSC2 expression comparison
  - `5B_Within_Context_Subtypes/` - TUSC2 analysis within each tissue context
  - `5C_DEG_TUSC2_Binary/` - Differential expression analysis

### 6_Cross_Context_Synthesis
- **Purpose**: Comparative analysis across all contexts
- **Subdirectories**:
  - `TUSC2_Functional_Impact_Comparison/` - Cross-context functional impact
  - `TUSC2_Impact_on_Subtype_Programs/` - Subtype program analysis
  - `TUSC2_Impact_on_Dev_State/` - Developmental state analysis

## Standard Subdirectory Structure

Each main analysis directory typically contains:

### figures/
Organized by analysis type:
- `context_markers/` - Context-specific marker plots
- `functional_signatures/` - Functional signature heatmaps and dotplots
- `developmental_signature_dotplots/` - Developmental marker plots
- `neurotransmitter_receptor_signature_dotplots/` - Neurotransmitter receptor analysis
- `interleukin_downstream_signature_dotplots/` - Interleukin pathway analysis

### data_for_graphpad/
- Parallel structure to figures/ with CSV files for each plot
- Enables easy import into GraphPad Prism or other analysis software

### stat_results_python/
- Statistical test results
- Cross-tabulation tables
- DEG analysis results

### marker_lists/
- Gene lists for each subtype and analysis
- Text files with one gene per line for easy reuse

## Dynamic Directory Creation

The pipeline creates subdirectories as needed based on:
- Available data contexts (Blood, NormalTissue, TumorTissue)
- Analysis types being performed
- Tang dataset subset analysis (when applicable)

## File Naming Convention

Files follow a structured naming pattern:
```
P{Part}_{Section}_{PlotType}_{Description}_{Context}_{Version}
```

Example: `P2_3a_Heatmap_DevProfile_Blood_v5.png`

## Notes

- Directories are created automatically by the analysis pipeline
- Empty directories may exist if certain analyses are skipped due to data availability
- The structure is designed to support both automated analysis and manual review
- All output is designed to be publication-ready with consistent formatting
